Types of Keys in Database

1. Primary Key

A primary key is a column or set of columns in a table that uniquely identifies each row in the table.

Example: In a table of students, the student ID column can be the primary key.

Student ID Name Age
1 John 20
2 Jane 21
3 Bob 22

2. Foreign Key

A foreign key is a column or set of columns in a table that refers to the primary key of another table.

Example: In a table of orders, the customer ID column can be a foreign key that references the customer ID column in the customers table.

Order ID Customer ID Order Date
1 1 2022-01-01
2 2 2022-01-15
3 1 2022-02-01

3. Composite Key

A composite key is a primary key that consists of multiple columns.

Example: In a table of exam results, the combination of student ID and exam ID columns can be a composite key.

Student ID Exam ID Score
1 1 80
1 2 90
2 1 70

4. Candidate Key

A candidate key is a column or set of columns in a table that can uniquely identify each row in the table.

Example: In a table of employees, the employee ID column and the combination of first name and last name columns can be candidate keys.

Employee ID First Name Last Name
1 John Doe
2 Jane Doe
3 Bob Smith

5. Alternate Key

An alternate key is a column or set of columns in a table that can uniquely identify each row in the table, but is not the primary key.

Example: In a table of customers, the email column can be an alternate key.

Customer ID Email Name
1 johndoe@example.com John Doe
2 janedoe@example.com Jane Doe
3 bobsmith@example.com Bob Smith

6. Artificial Key

An artificial key is a column or set of columns in a table that is created to uniquely identify each row in the table, but is not a natural part of the data.

Example: In a table of products, a unique identifier column can be an artificial key.

Product ID Product Name Price
UUID-1234 Product A 10.99
UUID-5678 Product B 9.99
UUID-9012 Product C 12.99
